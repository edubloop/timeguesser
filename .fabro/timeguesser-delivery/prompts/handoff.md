Write or update the ticket handoff summary.

Before writing:

- Read the current ticket artifact chain (`ticket.md`, `spec.md`, `plan.md`, `review.md`) and verification context.
- Read `Related ADRs` only when `spec.md` or `plan.md` explicitly lists them; if they say `None`, do not read architecture docs.
- Read workspace-level policy only when the ticket or spec explicitly anchors to it.

Use the workspace PR handoff structure:

- What was built
- Hypotheses tested
- Verification
- Scope check
- Risks and open questions
- Recommended next steps

Keep it concise and review-ready.
